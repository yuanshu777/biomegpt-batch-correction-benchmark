# CRC BiomeGPT GRL Benchmark

This is a focused repair package for the final CRC/MMUPHin/BiomeGPT/GRL
experiment. It does not restore every historical BiomeGPT direction.

Scientific goal: test whether a BiomeGPT/scGPT-style representation correction
can reduce study/batch signal while preserving CRC/control signal.

## Benchmarks

Primary benchmark:

- 389 overlapping MMUPHin CRC samples matched to previous BiomeGPT metadata by
  public accession evidence.
- Compare raw abundance, MMUPHin-adjusted abundance, raw BiomeGPT CLS, and
  GRL-corrected BiomeGPT CLS.

Reference benchmark:

- Full 551-sample MMUPHin CRC dataset.
- Compare raw abundance versus MMUPHin-adjusted abundance only.

Do not use full-551 MMUPHin results as the main direct comparison against
389-sample BiomeGPT results.

## Current Local Workflow

Build the 389-sample overlap assets:

```bash
python scripts/build_crc_overlap_benchmark.py
```

Check available assets:

```bash
python scripts/check_assets.py
```

Attempt CLS extraction or report missing assets:

```bash
python scripts/extract_biogpt_cls_for_crc_overlap.py
```

Evaluate available method matrices:

```bash
python scripts/evaluate_crc_methods.py
```

Run smoke tests:

```bash
pytest tests/
```

## Checkpoints and Missing Assets

`taxonomy_checkpoint_stage2 (1).pt` is configured as the default BiomeGPT
checkpoint, and `taxonomy_checkpoint_stage1 (1).pt` is retained as an explicit
asset. See `reports/checkpoint_inventory.md`.

`scripts/extract_biogpt_cls_for_crc_overlap.py` has generated
`outputs/crc_overlap_benchmark/biogpt_raw_cls_389.csv` from the stage2
checkpoint using the local Miniconda Python environment with PyTorch. The
remaining expected missing artifact is GRL-corrected CLS, which should only be
created after the correction training step is run later.
